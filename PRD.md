# 03\. 电子哨兵 Admin 系统产品需求文档 PRD

# 产品定位

电子哨兵 Admin 系统是电子哨兵 Web 大屏的后台管理系统，用于支撑多组织、多角色、多设备、多批次、多告警的统一管理。

它与 Web 大屏、告警 APP 共同构成电子哨兵的“端侧节点 \+ 告警闭环平台”。

---

# 用户角色

1. **系统管理员 \(System Admin\)**

    - **角色定位**：平台的最高权限拥有者，通常为平台方也就是我们自己（NH\_Admin）。

    - **核心职责**：负责全局系统的基础配置、所有入驻组织的创建与管理、全局设备大盘的监控，以及系统级功能（如全局 OTA 升级推送）的管理。

    - **数据可见性**：全平台所有组织、所有设备、所有告警数据。

2. **组织管理员 \(Organization Admin\)**

    - **角色定位**：每个合作机构或者采购方的管理员。

    - **核心职责**：负责本组织内部的账号分配与管理、本组织名下设备的绑定与日常管理、批次管理，以及统筹管理本组织的告警处理情况。

    - **数据可见性**：仅限本组织的所有数据。

3. **组织用户 \(Organization User\)**

    - **角色定位**：各企业/机构内部的基层业务执行人员（如安保人员、监控室值班员等）。

    - **核心职责**：负责日常业务的执行与响应，主要查看设备状态、接收并处理告警事件。

    - **数据可见性**：仅限被分配或本组织内的业务数据，无管理权限。

## 功能模块总览

## 1、登录鉴权

### 需求说明

管理系统无需开放注册，由超级管理员创建账号并分配角色。登录鉴权与 Web 大屏保持一致，采用双 Token 无感刷新机制。

### 功能点

1. 登录：用户输入账号密码进行登录，账号支持双体系（账号和手机号），密码默认是复杂密码，避免被破解。

    1. 账户：nh\-admin/13011112222

    2. 密码：hfklhjs\_f233jjks@\!

2. 双 Token 刷新机制：登录成功后默认 24h \* 3 天无需重复登录，当用户在系统内部调用接口的时候，使用 Refresh Token 无限刷新，保持登录状态，避免每天重复登录导致用户体验下降。

3. 退出登录：清除 Token 返回到登录页

4. 权限路由：每个用户都有属于自己的权限，用户只能访问自己权限内的功能，其他页面无法访问，输入后直接 403 重定向到 Web 大屏首页。

## 2、组织管理

### 需求说明

超级管理员可以创建组织，例如：

- 新疆合作单位 01：XJA

- 西藏合作单位 01：XZA

- 新疆合作单位 02：XJB

组织管理员只能管理本组织，不能跨组织操作。

### 功能点

1. **新增组织**：系统管理员可创建新组织，需填写组织名称、所属地区、联系人、联系电话、详细地址等基础信息。提交后系统自动生成唯一的组织 ID（Org\_ID）。

2. **编辑组织**：支持修改已创建组织的基础信息，组织名称修改需进行全局唯一性校验。

3. **停用/启用组织**：支持对组织状态进行切换。组织被“停用”后，该组织下的所有用户将被强制下线且无法再次登录，该组织名下的设备暂停业务交互。

4. **组织列表查询**：支持按组织名称、联系人进行模糊搜索；支持按组织状态（启用/停用）进行筛选；列表需支持分页显示。

## 3、用户管理

### 需求说明

用户管理对于超级管理员来说是单独的页面，对于组织管理员来说不单独作为一级页面，而是嵌入组织详情中。

### 功能点

1. **新增用户**：管理员可创建子账号。需填写用户名、手机号（作为登录账号），并分配所属组织及用户角色（如组织管理员、组织用户）。保存后系统自动生成默认密码。

2. **编辑用户**：支持修改用户的基本信息及角色权限。若修改了用户角色，该用户下次登录或刷新页面时权限即刻生效。

3. **重置密码**：提供一键重置密码功能，管理员操作后，该用户密码恢复为系统默认密码（如 guardx\_pass1234\!）。

4. **停用/启用用户**：支持对单个用户账号进行冻结或解冻。停用的用户无法登录 Admin 系统及告警 APP。

5. **用户列表查询**：支持按用户名、手机号模糊搜索；支持按所属组织、用户角色、账号状态进行条件筛选及分页。

## 4、设备管理

### 需求说明

管理员是单独的设备管理页面，组织管理员是嵌入到组织详情页面，设备管理用于维护电子哨兵设备资产，包括 2\.0 AOV 设备和 3\.0 HVS 设备。

### 功能点 

1. **设备入库与绑定**：支持单台录入或批量导入设备。需输入设备 SN 码、MAC 地址，并选择归属组织进行绑定。SN 码需进行系统唯一性校验。

2. **设备信息编辑**：支持修改设备的业务属性，如设备别名、安装位置、备注信息等。

3. **设备解绑/删除**：支持将设备与当前组织解绑。解绑后设备状态变更为“未分配”，历史告警数据保留，但不再产生新业务数据。

4. **设备列表查询**：支持按设备 SN、设备别名模糊搜索；支持按在线状态（在线/离线）、归属组织进行筛选。

5. **设备详情查看**：点击进入详情页，可查看设备基础信息（如当前 IP、固件版本、网络类型、电量等）以及最近的运行日志。

### 设备列表字段

### 页面设计

#### **页面一：设备列表页**

本页面用于全局/组织内设备的日常管理与状态监控。

- **顶部操作区**：

    - **新增设备按钮**：点击跳转至“新增设备表单”页。

    - **批量导入/绑定按钮**：支持通过下载 Excel 模板，批量填入设备 ID 和基础配置进行快速绑定。

    - **导出按钮**：导出当前列表的设备数据。

- **搜索与筛选区**：

    - **设备搜索**：支持按设备 ID（SN 码）、设备名称模糊搜索。

    - **状态筛选**：下拉选择在线状态（全部/在线/离线/未激活）。

    - **组织筛选**：系统管理员专属，可下拉选择或搜索特定组织查看其设备；组织管理员默认仅看本组织。

- **数据列表字段**：

    - **设备 ID**：系统唯一标识（如 GuardXV1202605\_0001）。

    - **设备名称**：业务别名（如“南门主通道哨兵”）。

    - **归属组织**：显示绑定的组织名称。

    - **在线状态**：绿点表示“在线”，灰点表示“离线”。

    - **电量/网络**：3\.0 HVS 特有，直观展示当前电量百分比及网络类型（4G/WiFi）。

    - **最后心跳时间**：设备最近一次上报状态的时间。

    - **操作列**：包含“查看详情”（进入页面三）、“编辑”（修改名称/位置等）、“解绑”（解除设备与组织的绑定关系）。

#### **页面二：新增设备表单（设备绑定页）**

由于设备 ID 已在“批次管理”中提前生成，此处的“新增”本质上是\*\*“激活并绑定”\*\*设备到具体业务场景中。

- **表单字段**：

    - **设备 ID（必填）**：文本输入框。用户需输入从批次管理中获取的可用设备 ID。系统需实时校验该 ID 是否存在且处于“未绑定”状态。

    - **设备名称（必填）**：文本输入框。为设备起一个易于识别的业务名称。

    - **归属组织（必填）**：下拉选择框。系统管理员需手动选择分配给哪个组织；组织管理员操作时，该字段默认锁定为本组织。

    - **安装位置（选填）**：文本输入框或地图选点。记录设备的物理部署位置，便于后期维护和告警定位。

    - **设备备注（选填）**：文本域。用于记录安装人员、特殊配置要求等信息。

- **交互逻辑**：

    - 提交表单时，系统校验设备 ID 合法性。若 ID 错误或已被其他组织绑定，提示“设备 ID 无效或已被占用”。

    - 绑定成功后，设备状态由“未绑定”变更为“离线”（等待设备实际上电联网后转为“在线”），并自动返回设备列表页。

#### **页面三：设备详情页**

本页面用于全方位展示单台设备的运行状况、历史轨迹及配置信息，采用模块化卡片设计。

- **模块一：基础信息**

    - 展示设备 ID、设备名称、归属组织、硬件型号、所属批次、绑定时间、安装位置。

    - 提供“编辑”按钮，允许修改名称和位置。

- **模块二：实时状态**

    - **连接状态**：当前在线/离线状态、持续在线/离线时长。

    - **网络信息**：当前 IP 地址、网络类型（4G/Wi\-Fi/以太网）、实时信号强度（如 \-75dBm 或信号格数）。

    - **硬件状态**：当前剩余电量（%）、充电状态（放电中/充电中）、当前固件版本号。

- **模块三：位置信息**

    - 静态展示填写的安装地址。

    - （可选）若设备带 GPS 模块或支持坐标录入，可嵌入小型地图组件展示设备定位。

- **模块四：最近告警**

    - 列表展示该设备最近触发的 5\-10 条告警记录。

    - 字段包括：告警时间、告警类型、处理状态。

    - 提供“查看更多”快捷链接，点击带上当前设备 ID 参数跳转至【告警列表管理】页面。

- **模块五：设备日志**

    - 按时间倒序展示设备的运行流水。

    - 包括：上线/下线记录、配置下发记录、主动唤醒记录等，便于技术排查问题。

- **模块六：OTA 记录（建议二期）**

    - 展示该设备的历史升级记录。

    - 字段包括：升级时间、升级前版本、升级后版本、升级状态（成功/失败/回滚）。

## 5、批次管理

### 需求说明

仅超级管理员可访问，批次用于管理设备生产、入库、交付和组织分配。

### 功能点

1. **新建批次**：用户可创建新的布控/管理批次，需填写批次名称、生效时间段，并勾选关联的具体设备清单。

2. **编辑批次**：支持在批次未结束前，修改批次的生效时间或增删关联的设备。

3. **批次启停控制**：支持手动“启动”或“结束”批次任务。结束后的批次归档为历史记录，不可再修改。

4. **批次列表查询**：支持按批次名称搜索；支持按批次状态（未开始、进行中、已结束）进行筛选查看。

### 页面设计

#### **页面一：批次列表页**

本页面用于全局查看和管理所有已创建的设备批次，掌握各批次设备的消耗（绑定）进度。

- **顶部操作区**：

    - **新增批次按钮**：点击后跳转至“新增批次页”（或弹出新增表单）。

    - **导出按钮**：支持将当前列表的批次统计数据导出。

- **搜索与筛选区**：

    - **批次编号搜索**：支持输入批次前缀模糊查询。

    - **创建时间筛选**：支持选择时间范围。

- **数据列表字段**：

    - **批次编号**：如 GuardXV1202605。

    - **设备总数**：该批次生成的设备 ID 总量（如 50）。

    - **已绑定数**：该批次下已被组织管理员绑定的设备数量。

    - **未绑定数**：该批次下剩余可用的设备 ID 数量。

    - **创建时间**：批次生成的具体时间。

    - **操作列**：包含“查看详情”（点击进入页面三）、“删除”（仅当该批次下所有设备均未绑定时允许删除，防止误删已投入使用的资产）。

#### **页面二：新增批次页（表单）**

本页面用于录入采购批次信息，系统将基于此表单自动生成对应数量的设备 ID 池。

- **表单字段**：

    - **批次编号（必填）**：文本输入框。作为设备 ID 的前缀，需限制仅支持英文、数字及下划线（如 GuardXV1202605）。系统需校验全局唯一性，不可与历史批次重复。

    - **设备数量（必填）**：数字输入框。输入该批次采购的设备总台数（如 50）。建议设置单次生成的最大上限（如 1000），防止恶意操作导致系统卡顿。

    - **设备型号（必填/选填）**：下拉选择框。选择该批次设备的硬件型号（如 V1\.0、V2\.0），便于后续固件 OTA 区分。

    - **备注说明（选填）**：文本域。用于记录采购合同号、供应商信息或其他备注。

- **交互逻辑**：

    - 用户填写完毕点击“确认生成”后，系统后台自动执行循环，生成指定数量的设备 ID。

    - **ID 生成规则**：批次编号 \+ 下划线 \+ 4位连续流水号（例如输入批次 GuardXV1202605，数量 50，则系统自动生成 GuardXV1202605\_0001 至 GuardXV1202605\_0050 的 50 条设备基础数据）。

    - 生成成功后，提示“成功生成 50 个设备 ID”，并自动返回“批次列表页”。

#### **页面三：批次详情页（设备状态明细）**

本页面用于查看某一特定批次下，所有自动生成的设备 ID 的具体去向和当前状态。

- **顶部基础信息区**：

    - 展示当前批次的核心信息：批次编号、设备型号、设备总数、已绑定数、未绑定数、创建时间、备注。

    - **导出 ID 按钮**：支持一键导出该批次下的所有设备 ID 列表（Excel 格式），方便线下分发给各组织管理员去进行绑定操作。

- **列表搜索区**：

    - **设备 ID 搜索**：支持输入尾号或完整 ID 搜索。

    - **绑定状态筛选**：下拉选择“全部 / 已绑定 / 未绑定”。

    - **归属组织筛选**：下拉选择系统内的组织，查看该批次分配给了哪些组织。

- **设备明细列表字段**：

    - **设备 ID**：系统自动生成的唯一标识（如 GuardXV1202605\_0001）。

    - **绑定状态**：高亮标签显示“已绑定”或“未绑定”。

    - **归属组织**：若已绑定，显示对应的组织名称；若未绑定，显示“\-”。

    - **在线状态**：若已绑定且设备已激活，显示“在线”或“离线”；未绑定则显示“\-”。

    - **绑定时间**：该设备 ID 被组织管理员绑定的时间。

    - **操作列**：包含“查看设备”（点击可跳转至【设备管理】模块的该设备详情页，仅对已绑定的设备展示）。

## 6、告警列表管理

### 需求说明

告警列表用于管理电子哨兵产生的告警事件，并支撑“告警 → 核查 → 处置 → 复盘”闭环。

### 功能点

1. **告警列表查询**：支持按时间范围、设备名称/SN、告警类型、告警级别进行多维度复合筛选；支持按处理状态（待处理、处理中、已处理）进行分类查看。

2. **告警详情查看**：支持查看单条告警的详细上下文，包括：抓拍图片、告警短视频/动图、触发时间、触发设备及位置信息。

3. **告警数据导出**：支持将当前筛选条件下的告警列表导出为 Excel/CSV 文件，供线下汇报与存档使用。

### 告警列表字段

### 页面设计

#### **页面一：告警列表页**

本页面是安保人员和管理员日常监控、查询和跟进告警事件的核心工作台。

- **搜索与筛选区**：

    - **时间范围**：支持选择“今日、近 7 天、近 30 天”或自定义起止时间（精确到时分秒）。

    - **设备搜索**：支持输入设备名称或设备 ID（SN 码）进行精确/模糊查询。

    - **告警类型**：下拉选择（如：人员闯入、未戴安全帽、设备破坏、异常徘徊等，根据实际 AI 算法定义）。

    - **告警级别**：下拉选择（如：严重、警告、提示）。

    - **处理状态**：单选/Tab 切换（全部 / 待处理 / 处理中 / 已处理）。

    - **归属组织**：系统管理员专属筛选条件。

- **操作区**：

    - **导出按钮**：将当前筛选条件下的列表数据导出为 Excel（包含告警基础信息及处理结果文本，不含视频/图片实体）。

- **数据列表字段**：

    - **告警编号**：系统生成的唯一流水号。

    - **告警快照：**照片缩略图

    - **触发时间**：告警发生的精确时间（YYYY\-MM\-DD HH:MM:SS）。

    - **告警类型**：级别建议用颜色区分（如人、车、无人机、人\|车、人\|无人机、车\|无人机、人\|车\|无人机）。

    - **触发设备**：显示设备名称及设备 ID。

    - **位置信息**：设备绑定的安装位置。

    - **处理状态**：高亮标签（待处理\-红色 / 处理中\-黄色 / 已处理\-绿色）。

    - **处理人**：若已接单或已处理，显示对应操作人员的姓名/账号。

    - **操作列**：

        - 查看，点击查看大图

        - 删除，点击删除，需要二次确认

## 7、OTA 管理，建议二期

### 需求说明

OTA 用于远程升级设备固件、算法模型、配置文件。

### 功能点

1. **固件包管理**：系统管理员可上传新的设备固件包，需填写版本号、MD5 校验值、更新说明（Release Notes）。

2. **升级任务下发**：支持按设备 SN、按组织、或按批次定向下发 OTA 升级指令。支持选择“立即升级”或“定时夜间升级”。

3. **升级状态追踪**：实时监控并展示每台设备的升级进度与结果状态（如：待下载、下载中、升级中、升级成功、升级失败）。针对失败设备提供“重新下发”按钮。

## 8、主动唤醒，建议二期

### 需求说明

主动唤醒用于远程唤醒 HVS 设备进行状态采集、拍照或配置更新，如切换 DVS 工作时间、切换图片/视频等功能。

### 功能点

1. **单设备主动唤醒**：在设备列表或详情页，针对处于“休眠”状态的设备，提供“主动唤醒”操作按钮。点击后系统通过长连接或信令通道向设备下发唤醒指令。

2. **唤醒结果反馈**：指令下发后，前端展示 Loading 状态。若在规定时间（如 15 秒）内收到设备上线心跳，则提示“唤醒成功”并刷新设备状态；若超时未响应，则提示“唤醒失败，请检查设备网络或电源”。
